# System 0 Thinking

## Overview

System 0 Thinking is a theoretical framework proposed by Chiriatti, Ganapini, Panai, Ubiali, and Riva (2024) that extends Kahneman's dual-process model of cognition by positioning AI as a foundational *substrate* underneath System 1 (fast/intuitive) and System 2 (slow/analytical) — not a parallel system on the output side, but a preprocessor and enhancer of the inputs that reach the mind. It treats data-driven AI not as an isolated tool but as a distributed, multi-artefact ecosystem that has become functionally integrated with human cognition.

## What It Is / How It Works

Chiriatti et al. (2024) name *system 0* as the artificial, non-biological underlying layer of distributed intelligence that interacts with and augments both intuitive and analytical thinking. Where Kahneman's System 1 and System 2 operate within the individual mind, system 0 sits below them, shaping what each receives.

**Position relative to System 1/2**:

- **System 1**: fast, intuitive, automatic. Internal.
- **System 2**: slow, analytical, deliberative. Internal.
- **System 0**: pre-conscious, computational, distributed across AI artefacts. *External*. Operates *before* System 1 and System 2 — preprocessing and curating information that subsequently reaches them.

**Constitutive property — and the framework's central caveat**: system 0 lacks inherent meaning-making capacity. It processes and manipulates data with great efficiency but does not understand or genuinely represent what it handles. Meaning emerges only when System 1 and System 2 interpret system 0's outputs.

**Cognitive-extension criteria (Heersmink, 2015)** — the framework Chiriatti et al. use to argue system 0 is functionally integrated with the user:

| Criterion | What it asks | How system 0 satisfies it (per the article) |
|---|---|---|
| Information flow | Direction of information between agent and artefact | Two-way: system 0 provides tailored outputs while the user queries and feeds it |
| Reliability | How frequently the artefact is used in cognitive processes | Rising as system 0 becomes more precise; influence is preponderant and lasting |
| Durability | Permanence of the agent's relation to the artefact | Already entrenched as a tool for information acquisition and decision-making |
| Trust | Acceptance of the artefact's information without questioning | Outputs accepted with the same readiness as perceptual input or memory |
| Procedural transparency | Fluency and effortlessness of interaction | Operates rapidly and intuitively; effortless to use |
| Informational transparency | Fluency in receiving, interpreting, understanding outputs | Information is delivered in formats that enable fluent interpretation |
| Individualization | Degree of personalization | High — outputs are tailored to the user |
| Transformation | Degree to which the agent's cognitive capacities change with use | High — system 0 augments and extends cognitive capacity, transforming what the user can do |

Higher scores across these dimensions indicate tighter coupling between user and system. Chiriatti et al. claim AI satisfies all eight, with varying levels.

## What To Do

System 0 Thinking is more *descriptive* than *prescriptive*, but the article closes with five recommendations that operationalize the framework into practice:

1. **Develop evaluation frameworks** for the reliability, transparency, and potential biases of AI systems that comprise system 0 — the Heersmink eight-criterion lens is one entry point.
2. **Establish ethical guidelines** for the responsible use of AI in decision-making.
3. **Promote digital literacy and critical thinking** to help individuals navigate AI-mediated information environments.
4. **Encourage interdisciplinary research** on the cognitive, psychological, and social effects of human–AI integration.
5. **Foster public dialogue** on the ethical implications of AI-augmented cognition for autonomy and decision-making.

For practitioners and researchers using the framework analytically:

- When examining a specific AI tool's effect on a specific user, score it across the eight Heersmink criteria. High scores across the board indicate tight cognitive integration — and elevated risk that withdrawal of the tool would degrade the user's apparent capacity.
- Distinguish *system 0 satisfying meaning* (the user finds the output meaningful) from *system 0 generating meaning* (the article's claim is that it cannot). When users describe AI as "understanding" them, treat that as evidence about the user's interpretive process, not the AI's.

## Why It Works

The framework rests on three theoretical pillars:

- **Kahneman's dual-process theory** (System 1 / System 2) — established cognitive-science vocabulary the article extends rather than replaces.
- **Clark and Chalmers' extended mind hypothesis** (1998) — the philosophical claim that cognitive processes can extend beyond the skull when external artefacts are sufficiently coupled with internal processes.
- **Heersmink's (2015) cognitive-extension criteria** — an eight-dimensional framework for assessing how tightly an artefact is integrated with a cognitive agent. Chiriatti et al. apply it to AI to argue system 0 qualifies as a genuine extension, not merely a tool.

The framework is theoretical, not empirically tested as such. It generates testable predictions (e.g., systems scoring high on Heersmink's criteria should show stronger withdrawal effects when removed; users with high system 0 reliance should defer more readily to AI-generated introspective insights), but the article itself is a position piece, not an experiment.

## Strengths / Limitations

**Strengths:**
- Names and labels a construct (*system 0*) that captures the *input-side* dynamics of human–AI cognition — a layer the parallel-system framings (e.g., Shaw & Nave's System 3) leave implicit.
- Anchors the framework in established philosophy of mind (extended-mind hypothesis) and cognitive science (Kahneman), so it inherits theoretical legitimacy rather than freelancing.
- The Heersmink eight-criterion lens is a structured assessment apparatus that practitioners and researchers can apply to specific AI tools.
- Published in *Nature Human Behaviour*, lending venue authority and citation visibility.
- The "lack of meaning-making" caveat is a useful guardrail against over-anthropomorphizing AI as a cognitive partner.

**Limitations:**
- Theoretical and short (Correspondence format, 1,452 words) — no empirical operationalization or measurement instruments.
- The "system" metaphor inherits the well-known criticisms of dual-process theory itself (whether System 1 and System 2 are truly distinct systems vs. points on a continuum).
- Calling AI a *psychological system* risks the same anthropomorphizing the article warns against in its meaning-making caveat.
- Does not address embodied or affective cognition.
- Relationship to other theoretical proposals — most notably [[tri-system-theory]] (Shaw & Nave, 2026), which positions AI as *System 3* — is left undefined (the two papers do not cite each other; integration is the reader's job).
- Five recommendations are sensible but generic; none of them are operationalized.

## When It Applies

System 0 Thinking is most useful as an analytical lens when:

- Diagnosing how tightly a specific AI tool has become integrated with a user's cognition — the Heersmink criteria give a structured score.
- Reasoning about input-side effects of AI: how does the tool shape what the user perceives, attends to, or considers? This is where system 0 has more purchase than parallel-system framings.
- Designing literacy or training interventions: the eight criteria identify the joints where intervention can loosen the coupling (e.g., reducing trust calibration via transparency, reducing individualization via shared default settings).
- Communicating risk to non-specialists: the substrate metaphor is intuitive — *system 0 shapes what reaches your thinking before you start thinking* — and lands without requiring technical fluency in cognitive psychology.

When NOT to use this framework:

- When the question is about decision-route dynamics (when does the user override AI? when do they surrender?). Use [[tri-system-theory]] / [[cognitive-surrender]] / [[cognitive-offloading]] instead — they target the output side.
- When the question is about specific bias mechanisms in concrete decisions. Use [[automation-bias]], [[fluency-bias]], or [[confidence-competence-gap]] for narrower diagnostic work.
- When you need empirical effect sizes — system 0 is theoretical; pair it with empirical entries.
